# Bundle reports

Track the web bundle over time. Each snapshot is a standalone HTML treemap
generated by `rollup-plugin-visualizer` — open in any browser.

## Generating a fresh report

```bash
SEALD_VISUALIZE=1 pnpm --filter web build
cp apps/web/dist/stats.html docs/bundle/YYYY-MM-DD-<milestone>.html
```

## 2026-04-24 — recipient signing flow

File: `2026-04-24-recipient-flow.html`

Chunk breakdown (gzipped, from `pnpm --filter web build`):

| Chunk | Size (kB, gzip) | Notes |
|---|---:|---|
| `index-*.js` (main) | 349.7 | sender + auth + dashboard + contacts |
| `pdf.worker.min-*.mjs` | n/a | pdfjs worker; loaded on demand by pages that render PDFs |
| `index-*.js` (largest split) | 8.1 | signer fill page (lazy) |
| Remaining signer pages | 1–2.4 each | all lazy-loaded |
| `RecipientHeader-*.js` | 1.3 | shared L2 chunk |
| `SignatureMark-*.js` | 0.6 | shared L1 chunk |
| `session-*.js` | 0.7 | signer session context |

**Result:** recipients who click an email link download only the signer
lane (`index-*.js` main for react + router + theme, plus the lazy page
chunk for whichever stage they land on). The sender surfaces stay in the
main bundle but aren't required for the signing flow — they're imported
only from the authed routes, so tree-shaking keeps them out of the
recipient's critical path on initial paint of `/sign/:id`.

**Follow-ups** (next bundle milestone):
- Split `react-router` into its own chunk (currently bundled into main).
- Split `@tanstack/react-query` into its own chunk (reused by both lanes).
- Consider moving the sender lane behind a top-level `lazy` boundary so
  recipients never pay for Supabase/Dashboard code even in the main chunk.
